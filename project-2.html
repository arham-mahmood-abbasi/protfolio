MS Decision Review System

This page details the case study of the Melbourne Storm project, including its overview, the tools used, and the significant impact it had.

Project Overview

On the MS project, I was responsible for developing several critical core features and implementing unique interactive visualizations. I developed robust systems for academy players, involving designing data structures to manage player profiles, training progress, performance metrics, and historical data. I also developed detailed player comparison views, requiring careful consideration of data aggregation, filtering, and presentation.

A particularly interesting task was implementing importing depth charts using Puppeteer, which involved automated browser interactions to scrape, parse, and integrate external data into our database, crucial for keeping player data up-to-date.

One of the most unique and technically challenging aspects was implementing a dynamic display of players on a field utilizing HTML Canvas. This involved creating an interactive visualization where players could be positioned, moved, and their attributes displayed in real-time. The challenge lay in precisely mapping player coordinates, handling various field layouts, and ensuring smooth animations and responsiveness. This feature required a deep understanding of canvas API intricacies, rendering performance, and geometric calculations.

The MS project presented several technical puzzles, and I successfully resolved the one concerning accurately displaying players on a canvas, showcasing strong problem-solving skills in interactive web development. This specific challenge required researching various canvas rendering techniques, experimenting with coordinate systems, and optimizing drawing operations. This persistence, combined with seeking guidance, allowed me to navigate these complex technical hurdles effectively.
